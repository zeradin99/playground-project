## 시나리오

각 시나리오는 Given / When / Then으로 작성. happy path와 high-cost error branch를 모두 덮는 만큼 만든다.

### 1. 시나리오 제목
- **Given** — 사용자나 시스템의 초기 상태
- **When** — 사용자가 취하는 행동
- **Then** — 관찰 가능한 결과

성공 기준:
- [ ] 구체적 입력 → 관찰 가능한 출력 (UI, API 응답, 저장 상태)
- [ ] 구체적 입력 → 관찰 가능한 출력

### 2. 시나리오 제목
- **Given** — ...
- **When** — ...
- **Then** — ...

성공 기준:
- [ ] ...

## 불변 규칙 (선택)

모든 시나리오에 걸쳐 유지되어야 하는 규칙. 범주 예: 보안/프라이버시, 성능, 데이터 일관성.

# Given / When / Then 작성 가이드

## 판정 기준

> "이 한 문장만 보고 개발자가 테스트를 쓸 수 있는가?"

No라면 시나리오가 너무 모호하거나, 관찰 가능한 행동이 아니라 구현을 기술하고 있다.

## 각 필드에 무엇을 쓰는가

| 필드 | 무엇을 쓰는가 | 허용 | 금지 |
|---|---|---|---|
| **Given** | 관찰 가능한 선행 조건: 화면 상태, 데이터 개수, 적용된 필터 | "할 일 3개가 표시된 리스트", "검색창에 'milk' 입력됨" | "todos.length === 3", "isLoading is true" |
| **When** | 구체적인 UI 요소와 그 위의 행동 | "'삭제' 버튼 클릭", "제목 필드에 'Meeting notes' 입력" | "deleteTodo() 호출", "SUBMIT 액션 디스패치" |
| **Then** | 컴포넌트 바깥에서 단언 가능한 결과 | "'할 일 2개' 텍스트가 표시됨", "빈 리스트 메시지 출현" | "todos.length === 2", "API가 호출됨" |

### 좋은 예 vs 나쁜 예

```
좋음:
  Given  제목 필드가 비어 있는 할 일 폼
  When   "추가" 버튼 클릭
  Then   "제목을 입력하세요" 에러 메시지가 표시된다

나쁨:
  Given  formState.title이 빈 문자열
  When   submit 이벤트 발생
  Then   유효성 에러가 설정된다
```

나쁜 버전으로 쓴 테스트는 컴포넌트 내부를 mock해야 하고, 구현을 바꾸면 깨지고, 사용자 행동에 대해 아무것도 증명하지 않는다.

## 성공 기준

각 항목은 **입력 → 관찰 가능한 출력** 한 쌍. 한 항목이 테스트 단언 하나(또는 소수)에 깔끔하게 매핑된다.

### 허용되는 출력

| 유형 | 예시 |
|---|---|
| 보이는 텍스트 | `"항목 추가됨"`이 나타난다 |
| 요소 개수 | 카드 3개가 렌더된다 |
| 요소 존재 | "삭제" 버튼이 보인다 |
| 필드 값 | 제목 필드에 `"우유 사기"` |
| 시각 상태 (클래스/속성으로 표현 가능) | 완료된 항목에 취소선 |
| API 응답 형태 (feature가 API를 노출할 때) | `200 { id: <new-id> }` |

### 금지되는 출력

| 유형 | 이유 |
|---|---|
| 내부 상태 (store, React state, signals) | 사용자가 관찰할 수 없다 |
| 함수 호출 단언 (`mockFn was called`) | 구현에 결합됨 — 리팩터링 시 깨진다 |
| 원시 DB 행, 서비스 계층 반환값 | 서버 내부. 사용자가 볼 수 없다 |

### 예시

```
좋음:
- [ ] title="우유 사기" → 텍스트 "우유 사기"인 항목이 추가되고 리스트 개수가 1이 된다
- [ ] 빈 제목 → 필드 아래 "제목을 입력하세요" 가 나타난다

나쁨:
- [ ] title="우유 사기" → todos 배열이 [{ id: 1, title: "우유 사기", done: false }] 가 된다
- [ ] title="우유 사기" → addTodo() 가 { title: "우유 사기" } 로 한 번 호출된다
```

## 불변 규칙을 쓸 때

3개 이상 시나리오에 거의 동일한 Then을 쓰고 있다면, 그 규칙은 불변 규칙에 속한다 — 모든 경로에 적용되므로 단일 G/W/T로 담을 수 없다.

| 범주 | 예시 |
|---|---|
| **보안 / 프라이버시** | 다른 사용자의 비공개 드래프트는 어떤 경로(UI, URL, 검색, RSS)로도 보이지 않는다 |
| **성능** | 모든 페이지가 4G에서 2초 안에 first contentful paint에 도달한다 |
| **데이터 일관성** | 표시되는 업보트 수는 기록된 투표 수와 항상 일치한다 |

이런 교차 규칙이 없는 feature라면 섹션을 통째로 생략한다.
